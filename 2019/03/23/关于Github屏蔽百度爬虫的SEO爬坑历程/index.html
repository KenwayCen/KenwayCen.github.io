关于Github屏蔽百度爬虫的SEO爬坑历程

发表于 2019-03-23 | 阅读次数：

我要一步一步往上爬，在最高点乘着叶片往前飞~

话说这个自己博客SEO 的坑源远流长，如果不是选择 Github，就不会存在他们高级玩家屏蔽和被屏蔽的事而影响，直到昨天才将 SEO 相关的基础配置弄好，可谓是效率极低了😂

下面就先给一个最终的解决方案的图，这只是一个方案，虽然我是这么处理，但它真的只是个方案，只要你清楚相关的为什么你就可以自己想怎么弄都行。

思路


如上图所示：当blog 元数据编写好后，hexo d 发布配置两个 git 仓库，分别推送发布到 github pages 仓库和 coding pages 仓库。以上就是爬坑后的整理，是不是很简单，是的，但是就是爬了好久才爬出来的。所以出来后才感叹，如果一开始没有将 blog 的推送到 github pages 仓库就不会出现这种问题了😂😂😂。为什么？因为 github 会屏蔽百度的爬虫，所以你的 blog 要让百度录入是不可能的。

正经讲一下解题思路

以下是针对一开始只放在 github pages 的情况来说的。若没有此情况，则以下对你来说都是废话。

1、注册 coding

现在 coding 和腾讯云绑在一块的，用微信登录就好，唯一需要注意的点是：

  • 绑定邮箱界面（可能是我理解有偏差，没错，就是那个界面）

就是这个界面

以上的输入框邮箱和密码半毛钱关系都没有，邮箱是邮箱，密码是 coding 登录密码，没有登录密码的话需要先设置登录密码。
我是在这个界面输入邮箱和邮箱密码，然后被提示好多次密码不对，是的，就是密码不对，我都输到自我怀疑了。
内心 n 个马跑过，为什么绑定邮箱需要邮箱密码，别人都不需要。emmm… 等等，对啊，这个压根就不是邮箱的密码😂😂😂😂

2、新建一个仓库

coding 是一个仓库就可以对应一个 pages 服务的，这个随意。

3、增加推送致 coding pages 仓库的 deploy

4、配置域名 DNS 解析

这里比较坑。尝试了很多次，而且时间消耗很大，因为 coding 自定义域名申请 有时间限制 ，30分钟申请一次，如果一次申请失败，就要等30分钟。满屏的心累。

  • 域名添加 CNAME 解析

我是在腾讯云买的域名，进入域名管理后台，对www.weilni.cn域名添加一条 CNAME 解析记录

注意：

1、记录值：静态：pages.coding.me

动态：pages.coding.io

2、线路：默认

3、暂停github pages（海外）服务的解析

4、等 coding pages 那边解析通过后 再把线路换成国内，再开启github pages服务（海外）的解析

  • coding 自定义域名

如果域名添加 DNS 解析时没有弄错，这里 coding 的自定义域名添加后，很快就可以申请通过，显示正常了。

至此，坑爬出来了。此时就可以去百度提交链接了。我主要是处理了包括主动提交和 sitemap文件的方式，详情请看 《GithubHexo博客SEO优化》


至于何时百度会完成收录，这个不一定，有说两天，有说两周，我现在就是时间问题了，如果实在太久，也可以和网友说的那样，到时候向百度反馈情况

ps: 为什么选 coding ？ 之前国内的 git 有用过码云，也知道码云也有 pages 服务，但是我想用它的时候发现码云 自定义域名是要收费的，基于能省则省的原则，用了 coding😎
