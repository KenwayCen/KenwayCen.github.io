Hexo 在新机器上继续写博客

发表于 2018-10-12 | 阅读次数：

很多时候我们搭建一个 Hexo 框架写博客，我们基本上都会在搭建的机器进行后续的博客编写。但是，假如以后换了一台新机器，或者原来写博客的机器被不可名状的系统更新后或者环境更新后出错了，写不了，是否需要考虑将原来博客的风格以及个性主题保留下来呢？所以就有了这篇文章。关于 Hexo的首次搭建相关的网上已经有很多相关讨论了，至于后续的这种更新迭代框架使用比较缺少，在这里记录一下自己的摸索后的相关资料内容，仅供本人参考。
在第一次搭建好 hexo 后，需要将站点的以下文件保存（上传到 github 上）

我之前的 github 博客搭建中已经将 博客的元数据也上传至 github 同一个仓库下的独立分支内方便去到新环境新机器的时候可以保证博客相关的配置。
主题也进行个性化处理放到了 github 上
化十博客的主题

所以当在新机器里编写博客的时候就需要进行以下几个步骤了。

博客元数据下载

  • clone github 上的博客元数据
  • 进入 themes/next 文件夹，将化十博客的主题也 clone 下来 放到 next里，ps:不要改 next目录的名称，只要将主题的所有内容放到 next 目录内。

    至此，博客的元数据已经完全准备好了，下面就是和第一次配置 hexo后写博客一样的过程了，需要简单的几个命令配置一下 hexo 的相关支撑插件
