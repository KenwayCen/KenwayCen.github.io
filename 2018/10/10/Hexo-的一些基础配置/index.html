Hexo 的一些基础配置

发表于 2018-10-10 | 阅读次数：

市面上那么多现成的可以写文章的网站，为什么选择 Hexo ？本人觉得原因有二：

  • 装逼
  • 简单而专注，你的眼里只有自己写的内容

好了，装逼完下面说说Hexo 主题的一些基础配置，主题的配置需要进入themes/next(以 next 主题为例)，打开_config.yml

设置菜单

菜单在 menu字段,默认是 home和archives。

  • 去掉不要的菜单
    例如，不想要archives，那就注释掉这个就好，在前面加个 #
  • 添加菜单
    这里的添加菜单是主题设置好的，只是没有放出来而已。至于自定义菜单，以后再研究。添加菜单只要把菜单前面那个 #去掉就好。例如：

    添加一个 about 菜单，则吧 about菜单前面的 # 去掉。

    注意: next主题默认是home和archives菜单,而 about 菜单是放在/about目录下的，但是默认这个目录是没有的，所以在添加 about菜单后，要创建文章（页面）时需要加一个 page 作为 layout

这样就可以看到有 about 的页面了

去掉底部的 copyright 中的 Hexo 信息

在主题的_config.yml ,有一个 copyright字段,将powered和theme的相关配置设置 为 false

设置语言

在站点的主配置文件_config.yml(不是主题的配置文件)配置language字段，将值设置成简体中文：

ps: 注意字段与值之间的空格

设置站点名称

在站点的主配置文件_config.yml(不是主题的配置文件)配置 title字段

设置作者

在站点的主配置文件_config.yml(不是主题的配置文件)配置 author 字段

设置部署环境

在站点的主配置文件_config.yml(不是主题的配置文件)配置deploy字段

  • type: 部署到github,写git
  • repo: 这个是存放发布内容的仓库，即github 上博客的托管仓库的 ssh url
  • branch: 这个是git 托管仓库的分支,本人是 master

注意： 所有是这配置文件里相关配置 都需要 在字段后 加个 英文状态的 空格
